[매개변수]

HttpServletRequest
HttpServletResponse
HttpSession

@RequestParam (생략가능)
파라미터 name과 동일한 이름의 변수를 매개변수로 작성하면
파라미터값을 쉽게 받을 수 있다

Model
request.setAttribute(name, value) 를 
model.addAttribute(name, value) 로 대체한다

커맨드 객체 : 클래스를 만들어서, 클래스의 멤버 필드의 값을 파라미터의 값으로 채워넣을 수 있다

HashMap : 파라미터 이름을 key, 파라미터 값을 value로 저장한다

PathVariable : 요청 주소상의 특정 구간을 파라미터로 처리하여 받을 수 있다



[반환형]

void : 요청 주소와 동일한 jsp를 찾는다
String : 직접 viewName을 지정한다
ModelAndView : viewName과 model을 동시에 수행하는 객체

mav.setViewName(String viewName)
mav.addObject(name, value)

ModelAndView mav = new ModelAndView(String viewName);
// viewName이 고정형태라면 생성자 매개변수로 처리할 수도 있다
// 제어문 흐름에 따라 viewName이 변동될 수 있다면
// mav.setViewName(String viewName) 을 활용하여 유동적으로 처리한다
// mav를 반환하면서 viewName을 아예 지정하지 않는다면, void와 동일하게 작동한다
